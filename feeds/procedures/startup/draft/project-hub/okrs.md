# [Platform] — OKRs (Objectives & Key Results)

**Project:** [Platform] (e-commerce marketplace)
**Document type:** Strategy — Quarterly goals
**Status:** 🚧 Draft (targets are placeholders — set real numbers as a team)

> **OKRs** turn the [roadmap](./roadmap.md) into measurable goals for a quarter. An **Objective** is a qualitative goal ("what we want"); **Key Results** are 3–5 measurable outcomes that prove we got there. Keep targets ambitious but few. Review at the [weekly meeting](./weekly-meeting.md); score at quarter-end (0.0–1.0).

---

## How to use this doc
- Set OKRs **per quarter**, aligned to the current [roadmap](./roadmap.md) phase.
- Each Objective gets an **owner** and 3–5 **measurable** Key Results.
- A KR must have a number and a target (e.g. "onboard **10** vendors"), not a task.
- Mid-quarter: track progress. Quarter-end: score each KR 0.0–1.0 and reflect.
