# Template: Risk / Issue / Change Log

> **How to use:** One living document, reviewed every sprint, per [06 — Risk, Issues & Change](../06-risk-issues-and-change.md). A risk is a future problem; an issue is a current one; a change request is a deliberate scope shift. Logging each visibly is how a PM keeps surprises off the table. Scope can change — *silent* change is the only sin.
