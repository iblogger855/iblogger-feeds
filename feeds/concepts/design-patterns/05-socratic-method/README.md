## бһҹбҹҒбһ…бһҖбҹ’бһҸбһёбһ•бҹ’бһҸбһҫбһҳ (Introduction)

The **Socratic Method** is an ancient dialogue-based technique popularized by the philosopher Socrates. It completely shifts the dynamic from a cold, passive lecture to a warm, active, and guided journey of discovery. By asking patient, highly-targeted questions rather than just dumping facts onto the learner, you walk hand-in-hand with them until they confidently realize the truth themselves. This empathetic learning approach removes technical intimidation, builds immense confidence, and guarantees deep, emotional memory retention because the learner experienced the "Aha!" moment themselves.

## бһҡбһјбһ”бһҳбһ“бҹ’бһҸбһ“бҹғбһҖбһ¶бһҡбһҠбҹ„бҹҮбһҹбҹ’бһҡбһ¶бһҷ (The Formula)

```
1. Ask an initial question whose answer points toward the concept.
2. Accept the learner's answer (correct or incorrect).
3. Ask a logical follow-up question that challenges or refines that answer.
4. With each turn, narrow the possible solution space.
5. Guide the learner to state the final correct solution themselves.
6. Confirm: "Yes! And that's exactly what [Concept X] does."
```

## бһ§бһ‘бһ¶бһ бһҡбһҺбҹҚбһҮбһ¶бһҖбҹӢбһҹбҹ’бһҸбҹӮбһ„бҹ– бһҖбһ¶бһҡбһҹбҹ’бһңбҹӮбһ„бһҡбһҖбһ”бҹ’бһҡбһ–бҹҗбһ“бҹ’бһ’ Binary Search (Practical Example)

### Explaining Binary Search through Dialogue (English)
> **Socrates:** *"You're looking for page 347 in a 700-page book. Do you start from page 1?"*  
> **Learner:** *"No, I'd open the book somewhere near the middle."*  
> **Socrates:** *"Why the middle?"*  
> **Learner:** *"Because if 347 is bigger or smaller than wherever I land, I instantly know which way to go."*  
> **Socrates:** *"And once you know that direction, what happens to the other pages?"*  
> **Learner:** *"I throw away half the book. I don't need to look at them."*  
> **Socrates:** *"Exactly. How many times can you cut a 700-page book in half before you reach exactly one page?"*  
> **Learner:** *"I guess about 10 times."*  
> **Socrates:** *"Yes! That is exactly what we call Binary Search. It takes O(log n) time."*

> **бһҹбһјбһҖбҹ’бһҡбһ¶бһҸбҹ–** *В«бһ–бһ·бһҸбһ”бҹ’бһҡбһ¶бһҖбһҠбһҺбһ¶бһҹбҹӢ! бһҸбһҫбһўбҹ’бһ“бһҖбһўбһ¶бһ…бһҖбһ¶бһҸбҹӢбһҹбҹҖбһңбһ—бҹ… бҹ§бҹ бҹ  бһ‘бҹҶбһ–бҹҗбһҡбһ“бҹҒбҹҮбһҮбһ¶бһ–бһ¶бһҖбҹӢбһҖбһҺбҹ’бһҸбһ¶бһӣбһ”бһ¶бһ“бһ”бҹүбһ»бһ“бҹ’бһҳбһ¶бһ“бһҠбһ„ бһҳбһ»бһ“бһ–бҹҒбһӣбһўбҹ’бһ“бһҖбһ‘бҹ…бһҠбһӣбҹӢбһ‘бҹҶбһ–бҹҗбһҡбһҠбҹӮбһӣбһ…бһ„бҹӢбһ”бһ¶бһ“?В»*  
> **бһҹбһ·бһҹбҹ’бһҹбҹ–** *В«бһ”бҹ’бһҡбһ бҹӮбһӣбһҮбһ¶ бҹЎбҹ  бһҠбһ„бһ”бҹүбһ»бһҺбҹ’бһҺбҹ„бҹҮбҹ”В»*  
> **бһҹбһјбһҖбҹ’бһҡбһ¶бһҸбҹ–** *В«бһҸбҹ’бһҡбһ№бһҳбһҸбҹ’бһҡбһјбһңбһҺбһ¶бһҹбҹӢ! бһ“бҹҒбҹҮбһ бһҫбһҷбһӮбһәбһҮбһ¶бһўбҹ’бһңбһёбһҠбҹӮбһӣбһҷбһҫбһ„бһ бҹ…бһҗбһ¶ Binary Search бһҖбҹ’бһ“бһ»бһ„бһҖбһ»бҹҶбһ–бҹ’бһҷбһјбһ‘бҹҗбһҡ бһҠбҹӮбһӣбһҠбҹҶбһҺбһҫбһҡбһҖбһ¶бһҡбһҖбҹ’бһ“бһ»бһ„бһҖбһҳбҹ’бһҡбһ·бһҸбһӣбҹ’бһ”бһҝбһ“ O(log n)бҹ”В»*

## бһҳбҹҒбһҡбҹҖбһ“ бһ“бһ·бһ„бһҠбҹӮбһ“бһҖбҹҶбһҺбһҸбҹӢ (When to Use & Limitations)

### рҹ“Ҳ Best For (бһҹбһ¶бһҖбһҹбһҳбһ”бҹҶбһ•бһ»бһҸбһҹбһҳбҹ’бһҡбһ¶бһ”бҹӢ)
* **1-on-1 Mentoring:** Guiding junior developers to debug their own errors without telling them the line to fix.
* **Code Reviews:** Asking questions like *"What happens if this input is null?"* to help them notice edge cases.
* **Technical Interviews:** Evaluating a candidate's problem-solving depth and adaptability.

### вҡ пёҸ Limitations (бһҠбҹӮбһ“бһҖбҹҶбһҺбһҸбҹӢ)
* **Requires Patience:** Takes longer than just telling the answer directly.
* **Interactive Only:** Hard to perform in static, one-way media (like pre-recorded videos), though highly effective in written parables.
* **Risk of Frustration:** If the questions are too vague, the learner may feel interrogated rather than guided. Keep questions friendly, encouraging, and highly targeted.

## рҹ“ҡ Implemented Patterns (бһӮбҹҶбһҡбһјбһҹбҹ’бһҗбһ¶бһ”бһҸбҹ’бһҷбһҖбһҳбҹ’бһҳбһҠбҹӮбһӣбһ”бһ¶бһ“бһўбһ“бһ»бһңбһҸбҹ’бһҸ)

Here are the design patterns explained using the interactive **Socratic Method** dialogue:

* **[01. Command (бһҖбһ¶бһҡбһ”бҹҶбһ”бҹ’бһӣбҹӮбһ„бһҹбҹҶбһҺбһҫбһҖбһ¶бһҡбһ„бһ¶бһҡбһ‘бҹ…бһҮбһ¶ Object бһҸбһ¶бһҳбһңбһ·бһ’бһёбһҹбһ¶бһҹбҹ’бһҸбҹ’бһҡбһҹбһјбһҖбҹ’бһҡбһ¶бһҸ)](./01-command.md)** вҖ” Guides a developer to discover that requests themselves must be treated as independent objects to support undo logs, scheduling, and multi-device triggers easily.
* **[02. Iterator (бһҖбһ¶бһҡбһҹбҹ’бһңбҹӮбһ„бһҡбһҖбһ‘бһ·бһ“бҹ’бһ“бһ“бҹҗбһҷбһҠбҹ„бһҷбһҹбһ»бһңбһҸбҹ’бһҗбһ·бһ—бһ¶бһ–бһҸбһ¶бһҳбһңбһ·бһ’бһёбһҹбһ¶бһҹбҹ’бһҸбҹ’бһҡбһҹбһјбһҖбҹ’бһҡбһ¶бһҸ)](./02-iterator.md)** вҖ” Guides a developer to discover that they must separate list/tree traversals into polymorphic helper objects (Iterators) to protect class encapsulation.
* **[03. Visitor (бһҖбһ¶бһҡбһ”бһ“бҹ’бһҗбҹӮбһҳбһҳбһ»бһҒбһ„бһ¶бһҡбһҠбҹ„бһҷбһҳбһ·бһ“бһҖбҹӮбһ”бҹ’бһҡбҹӮбһҖбһјбһҠбһ…бһ¶бһҹбҹӢбһҸбһ¶бһҳбһңбһ·бһ’бһёбһҹбһ¶бһҹбҹ’бһҸбҹ’бһҡбһҹбһјбһҖбҹ’бһҡбһ¶бһҸ)](./03-visitor.md)** вҖ” Guides a developer to discover Double Dispatch, letting them attach unlimited rendering or spell-checking plugins without modifying core document nodes.
* **[04. Singleton (бһҖбһ¶бһҡбһ”бһ„бҹ’бһҖбһҫбһҸбһ”бҹ’бһҡбһ–бҹҗбһ“бҹ’бһ’бһҖбһ¶бһҡбһ–бһ·бһҸбһҸбҹӮбһҳбһҪбһҷбһӮбһҸбҹӢбһҸбһ¶бһҳбһңбһ·бһ’бһёбһҹбһ¶бһҹбҹ’бһҸбҹ’бһҡбһҹбһјбһҖбҹ’бһҡбһ¶бһҸ)](./04-singleton.md)** вҖ” Guides a developer to discover that global shared configs or connection pools must be restricted to a single instance inside memory to prevent resource conflicts and state inconsistency.
* **[05. Builder (бһҖбһ¶бһҡбһ”бһ„бҹ’бһҖбһҫбһҸ Object бһҹбҹ’бһҳбһ»бһӮбһҹбҹ’бһҳбһ¶бһүбһҸбһ¶бһҳбһңбһ·бһ’бһёбһҹбһ¶бһҹбҹ’бһҸбҹ’бһҡбһҹбһјбһҖбҹ’бһҡбһ¶бһҸ)](./05-builder.md)** вҖ” Guides a developer to discover that complex object creation must decouple from its representation, creating transient builder checklists to guarantee both compile-time readability and strict immutability.
* **[06. Factory Method (бһҖбһ¶бһҡбһ”бһ„бҹ’бһҖбһҫбһҸ Object бһҸбһ¶бһҳбһҸбһҳбҹ’бһҡбһјбһңбһҖбһ¶бһҡбһҷбһәбһҸбһҷбҹүбһ¶бһңбһҸбһ¶бһҳбһңбһ·бһ’бһёбһҹбһ¶бһҹбҹ’бһҸбҹ’бһҡбһҹбһјбһҖбҹ’бһҡбһ¶бһҸ)](./06-factory-method.md)** вҖ” Guides a developer to discover that coupling to concrete types can be resolved by deferring the implementation of an abstract creation method to specialized subclasses.
