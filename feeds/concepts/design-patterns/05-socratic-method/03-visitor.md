## бҹЎ. бһҖбһ·бһ…бҹ’бһ…бһҹбһ“бҹ’бһ‘бһ“бһ¶бһҹбҹ’бһңбҹӮбһ„бһҡбһҖбһҖбһ¶бһҡбһ–бһ·бһҸ (The Socratic Dialogue)

### English
* **Socrates:** "Glaucon, imagine you have spent months perfectly crafting a complex XML Document structure containing delicate elements: `HeaderNode`, `ParagraphNode`, and `ImageNode`. If the client begs you to add a feature to export this document to JSON, what will you do to your precious nodes?"
* **Glaucon:** "I... I suppose I must tear them open and add an `exportToJSON()` method to every single Node class."
* **Socrates:** "I see. And what if next week, they demand a plain text renderer? And the week after, a spell-checker? Will you keep painfully cutting open your stable, peaceful Node classes to stitch in these unrelated operations?"
* **Glaucon:** "Oh no... My beautiful, clean Node classes will become horribly disfigured! They will swell with hundreds of lines of rendering and spell-checking logic that have absolutely nothing to do with holding data!"
* **Socrates:** "Exactly. And does it not break your heartвҖ”and the Open-Closed PrincipleвҖ”to constantly risk breaking your core classes every time a new fad feature is requested?"
* **Glaucon:** "It terrifies me. One small typo in a new spelling function could crash the entire document system that I worked so hard to stabilize."
* **Socrates:** "Then how do we grant these classes new abilities without ever touching their pure source code? What if we invite a 'Guest' to do the work? We define a `Visitor` interface. We add just one permanent, welcoming method to our Nodes called `accept(Visitor v)`. The Node simply opens the door and says to the guest, `v.visit(this)`, trusting the guest to do the right thing."
* **Glaucon:** "Wow... That is beautiful! It is Double Dispatch! If we welcome guests this way, the Node classes are locked and safe forever! If we need a JSON exporter, we just send a `JSONExportVisitor`. If we need spell-checking, we send a `SpellCheckVisitor`! The core structure remains pure, peaceful, and perfectly safe!"
* **Socrates:** "Splendid, Glaucon. You have protected your creation and discovered the Visitor Pattern."

### Khmer
* **бһҹбһјбһҖбҹ’бһҡбһ¶бһҸбҹ–** В«бһӮбҹ’бһӣбһјбһҒбһ»бһ“ бһўбһҫбһҷ! бһҹбҹ’бһҡбһҳбҹғбһҗбһ¶бһўбҹ’бһ“бһҖбһ”бһ¶бһ“бһ…бҹҶбһҺбһ¶бһҷбһ–бҹҒбһӣбһҡбһ¶бһ”бҹӢбһҒбҹӮбһҠбһҫбһҳбҹ’бһ”бһёбһ…бҹ’бһ“бҹғбһ”бҹ’бһҡбһҢбһ·бһҸбһҡбһ…бһ“бһ¶бһҹбһҳбҹ’бһ–бҹҗбһ“бҹ’бһ’бһҜбһҖбһҹбһ¶бһҡ XML бһҠбҹҸбһҹбҹ’бһҡбһҹбҹӢбһҹбҹ’бһўбһ¶бһҸбһҳбһҪбһҷ бһҠбҹӮбһӣбһҳбһ¶бһ“бһ’бһ¶бһҸбһ»бһ•бҹ’бһҹбҹҶбһҷбҹүбһ¶бһ„бһӣбһҳбҹ’бһўбһ·бһҸбһҠбһјбһ…бһҮбһ¶бҹ– `HeaderNode`, `ParagraphNode` бһ“бһ·бһ„ `ImageNode`бҹ” бһ”бҹ’бһҡбһҹбһ·бһ“бһ”бһҫбһҖбһјбһ“бһҖбҹ’бһҸбһёбһўбһ„бҹ’бһңбһҡбһҹбһ»бҹҶбһұбҹ’бһҷбһўбҹ’бһ“бһҖбһ”бһ“бҹ’бһҗбҹӮбһҳбһҳбһ»бһҒбһ„бһ¶бһҡбһ“бһ¶бҹҶбһ…бҹҒбһүбһҜбһҖбһҹбһ¶бһҡбһ“бҹҒбҹҮбһ‘бҹ…бһҮбһ¶ JSON бһҸбһҫбһўбҹ’бһ“бһҖбһ“бһ№бһ„бһ’бҹ’бһңбһҫбһўбҹ’бһңбһёбһ…бҹҶбһ–бҹ„бҹҮ Node бһҠбҹҸбһҳбһ¶бһ“бһҸбһҳбҹ’бһӣбҹғбһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖ?В»
* **бһӮбҹ’бһӣбһјбһҒбһ»бһ“бҹ–** В«бһҒбҹ’бһүбһ»бҹҶ... бһҒбҹ’бһүбһ»бҹҶбһ”бҹ’бһҡбһ бҹӮбһӣбһҮбһ¶бһҸбҹ’бһҡбһјбһңбһңбҹҮбһҖбһ¶бһҸбҹӢбһ”бһҫбһҖбһ–бһҪбһҖбһңбһ¶ бһ бһҫбһҷбһ”бһ“бҹ’бһҗбҹӮбһҳбһҳбһ»бһҒбһ„бһ¶бһҡ `exportToJSON()` бһ…бһјбһӣбһ‘бҹ…бһҖбҹ’бһ“бһ»бһ„ Class бһҡбһ”бһҹбҹӢ Node бһ“бһёбһҳбһҪбһҷбҹ—бһ бһҫбһҷбҹ”В»
* **бһҹбһјбһҖбҹ’бһҡбһ¶бһҸбҹ–** В«бһҒбҹ’бһүбһ»бҹҶбһҷбһӣбҹӢбһ бһҫбһҷбҹ” бһ…бһ»бҹҮбһ”бһҫбһ“бҹ…бһҹбһ”бҹ’бһҸбһ¶бһ бҹҚбһҖбҹ’бһҡбҹ„бһҷ бһ–бһҪбһҖбһӮбҹҒбһ‘бһ¶бһҳбһ‘бһ¶бһҡбһҳбһ»бһҒбһ„бһ¶бһҡбһ”бһ„бҹ’бһ бһ¶бһүбһўбһҖбҹ’бһҹбһҡбһ’бһҳбҹ’бһҳбһҸбһ¶? бһ бһҫбһҷбһҹбһ”бҹ’бһҸбһ¶бһ бҹҚбһ”бһ“бҹ’бһ‘бһ¶бһ”бҹӢбһ‘бҹҖбһҸ бһ‘бһ¶бһҳбһ‘бһ¶бһҡбһҖбһ¶бһҡбһ–бһ·бһ“бһ·бһҸбҹ’бһҷбһўбһҖбҹ’бһҒбһҡбһ¶бһңбһ·бһҡбһ»бһ‘бҹ’бһ’? бһҸбһҫбһўбҹ’бһ“бһҖбһ“бһ№бһ„бһ”бһ“бҹ’бһҸбһңбҹҮбһҖбһ¶бһҸбҹӢ Class Node бһҠбҹҸбһҹбҹ’бһ„бһ”бҹӢбһҹбҹ’бһ„бһ¶бһҸбҹӢбһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖбһ‘бһ¶бҹҶбһ„бһҲбһәбһ…бһ¶бһ”бҹӢ бһҠбһҫбһҳбҹ’бһ”бһёбһҠбҹҒбһҡбһ—бҹ’бһҮбһ¶бһ”бҹӢбһҖбһ·бһ…бҹ’бһ…бһҖбһ¶бһҡбһҠбҹӮбһӣбһҳбһ·бһ“бһ–бһ¶бһҖбҹӢбһ–бҹҗбһ“бҹ’бһ’бһ‘бһ¶бҹҶбһ„бһ“бҹҒбҹҮбһ¬?В»
* **бһӮбҹ’бһӣбһјбһҒбһ»бһ“бҹ–** В«бһўбһјбһ‘бҹҒ... Class Node бһҠбҹҸбһҹбҹ’бһҡбһҹбҹӢбһҹбҹ’бһўбһ¶бһҸбһҡбһ”бһҹбҹӢбһҒбҹ’бһүбһ»бҹҶбһ…бҹ’бһ”бһ¶бһҹбҹӢбһҮбһ¶бһҒбһјбһ…бһ‘бҹ’бһҡбһ„бҹӢбһ‘бҹ’бһҡбһ¶бһҷбһўбһҹбҹӢбһ бһҫбһҷ! бһңбһ¶бһ“бһ№бһ„бһЎбһҫбһ„бһ”бҹүбҹ„бһ„бһҠбҹ„бһҷбһҹбһ¶бһҡбһҖбһјбһҠбһҡбһ¶бһ”бҹӢбһҡбһҷбһҮбһҪбһҡбһ“бҹғбһҖбһ¶бһҡбһ”бһҖбһ”бҹ’бһҡбҹӮ бһ“бһ·бһ„бһҖбһ¶бһҡбһ–бһ·бһ“бһ·бһҸбҹ’бһҷбһўбһҖбҹ’бһҒбһҡбһ¶бһңбһ·бһҡбһ»бһ‘бҹ’бһ’ бһҠбҹӮбһӣбһӮбҹ’бһҳбһ¶бһ“бһ–бһ¶бһҖбҹӢбһ–бҹҗбһ“бҹ’бһ’бһўбҹ’бһңбһёбһ‘бһ¶бһӣбҹӢбһҸбҹӮбһҹбҹ„бҹҮбһ“бһ№бһ„бһҖбһ¶бһҡбһҡбһҖбҹ’бһҹбһ¶бһ‘бһ»бһҖбһ‘бһ·бһ“бҹ’бһ“бһ“бҹҗбһҷбһҹбҹ’бһ“бһјбһӣ!В»
* **бһҹбһјбһҖбҹ’бһҡбһ¶бһҸбҹ–** В«бһ–бһ·бһҸбһ”бҹ’бһҡбһ¶бһҖбһҠбһҺбһ¶бһҹбҹӢбҹ” бһ бһҫбһҷбһҸбһҫбһңбһ¶бһҳбһ·бһ“бһ’бҹ’бһңбһҫбһұбҹ’бһҷбһўбҹ’бһ“бһҖбһҲбһәбһ…бһ¶бһ”бҹӢбһ‘бҹҒбһ¬вҖ”бһ–бҹ’бһҡбһҳбһ‘бһ¶бҹҶбһ„бһӣбҹ’бһҳбһҫбһҹбһ“бһ№бһ„ Open-Closed Principle бһ•бһ„вҖ”бһҠбҹӮбһӣбһҸбҹ’бһҡбһјбһңбһ”бҹ’бһҡбһҗбһ»бһҷбһ”бҹ’бһҡбһҖбһ¶бҹҶбһ„бһҡбһ¶бһӣбҹӢбһӣбһҫбһҖбһҠбҹӮбһӣбһӮбҹҒбһ…бһ„бҹӢбһ”бһ¶бһ“бһҳбһ»бһҒбһ„бһ¶бһҡбһҗбҹ’бһҳбһёбһ“бҹ„бҹҮ?В»
* **бһӮбҹ’бһӣбһјбһҒбһ»бһ“бҹ–** В«бһңбһ¶бһ–бһ·бһҸбһҮбһ¶бһӮбһҪбһҡбһұбҹ’бһҷбһҒбҹ’бһӣбһ¶бһ…бһҺбһ¶бһҹбҹӢбҹ” бһҖбһ¶бһҡбһңбһ¶бһҷбһўбһҖбҹ’бһҹбһҡбһҒбһ»бһҹбһҸбҹӮбһҳбһҪбһҷбһҸбһҪбһ“бҹ…бһҖбҹ’бһ“бһ»бһ„бһҳбһ»бһҒбһ„бһ¶бһҡбһҗбҹ’бһҳбһё бһўбһ¶бһ…бһ“бһ№бһ„бһ”бҹҶбһ•бҹ’бһӣбһ¶бһүбһ”бҹ’бһҡбһ–бҹҗбһ“бҹ’бһ’бһҜбһҖбһҹбһ¶бһҡбһ‘бһ¶бҹҶбһ„бһҳбһјбһӣбһҠбҹӮбһӣбһҒбҹ’бһүбһ»бҹҶбһҒбҹҶбһ”бҹ’бһҡбһ№бһ„бһҸбһҳбҹ’бһӣбһҫбһ„бһҠбҹ„бһҷбһӣбҹҶбһ”бһ¶бһҖбҹ”В»
* **бһҹбһјбһҖбҹ’бһҡбһ¶бһҸбҹ–** В«бһ…бһ»бҹҮбһ’бҹ’бһңбһҫбһҠбһјбһ…бһҳбҹ’бһҸбҹҒбһ…бһ‘бһҫбһ”бһҷбһҫбһ„бһўбһ¶бһ…бһ•бҹ’бһҸбһӣбҹӢбһҹбһҳбһҸбҹ’бһҗбһ—бһ¶бһ–бһҗбҹ’бһҳбһёбҹ—бһҠбһӣбҹӢбһ–бһҪбһҖбһңбһ¶ бһҠбҹ„бһҷбһҳбһ·бһ“бһ”бһ¶бһ…бҹӢбһ”бҹүбҹҮбһ–бһ¶бһӣбҹӢбһҖбһјбһҠбһҠбҹҸбһ”бһҡбһ·бһҹбһ»бһ‘бҹ’бһ’бһҡбһ”бһҹбҹӢбһ–бһҪбһҖбһңбһ¶бһ‘бһ¶бһӣбҹӢбһҸбҹӮбһҹбҹ„бҹҮ? бһ…бһ»бҹҮбһ”бһҫ бһҷбһҫбһ„бһўбһүбҹ’бһҮбһҫбһү 'бһ—бҹ’бһүбҹҖбһң' бһұбҹ’бһҷбһҳбһҖбһҮбһҪбһҷбһ’бҹ’бһңбһҫбһҖбһ¶бһҡбһ„бһ¶бһҡбһ“бҹҒбҹҮ? бһҷбһҫбһ„бһ”бһ„бҹ’бһҖбһҫбһҸ Interface `Visitor` бһҳбһҪбһҷбҹ” бһ бһҫбһҷбһҷбһҫбһ„бһӮбҹ’бһҡбһ¶бһ“бҹӢбһҸбҹӮбһ”бһ“бҹ’бһҗбҹӮбһҳбһҳбһ»бһҒбһ„бһ¶бһҡбһҹбҹ’бһңбһ¶бһӮбһҳбһ“бҹҚбһўбһ…бһ·бһ“бҹ’бһҸбҹ’бһҡбҹғбһҷбҹҚбһҸбҹӮбһҳбһҪбһҷбһӮбһҸбҹӢбһ‘бҹ…бһҖбҹ’бһ“бһ»бһ„ Node бһӮбһә `accept(Visitor v)`бҹ” Node бһӮбҹ’бһҡбһ¶бһ“бҹӢбһҸбҹӮбһ”бһҫбһҖбһ‘бҹ’бһңбһ¶бһҡбһ бһҫбһҷбһ”бҹ’бһҡбһ¶бһ”бҹӢбһ‘бҹ…бһ—бҹ’бһүбҹҖбһңбһҗбһ¶ `v.visit(this)` бһҠбҹ„бһҷбһ‘бһ»бһҖбһ…бһ·бһҸбҹ’бһҸбһұбҹ’бһҷбһ—бҹ’бһүбҹҖбһңбһ’бҹ’бһңбһҫбһҖбһ¶бһҡбһ„бһ¶бһҡбһ“бҹ„бҹҮбҹ”В»
* **бһӮбҹ’бһӣбһјбһҒбһ»бһ“бҹ–** В«бһўбһјбһ бһј... бһ“бҹҒбҹҮбһ–бһ·бһҸбһҮбһ¶бһўбһҹбҹ’бһ…бһ¶бһҡбҹ’бһҷбһҺбһ¶бһҹбҹӢ! бһ“бҹҒбҹҮбһ бҹ…бһҗбһ¶ Double Dispatch! бһ”бһҫбһҷбһҫбһ„бһҹбҹ’бһңбһ¶бһӮбһҳбһ“бҹҚбһ—бҹ’бһүбҹҖбһңбһ”бҹӮбһ”бһ“бҹҒбҹҮ Class Node бһ“бһ№бһ„бһҸбҹ’бһҡбһјбһңбһ”бһ¶бһ“бһ…бһ¶бһҖбҹӢбһҹбҹ„бһҡбһҷбҹүбһ¶бһ„бһҳбһ¶бһ“бһҹбһ»бһңбһҸбҹ’бһҗбһ·бһ—бһ¶бһ–бһҮбһ¶бһҡбҹҖбһ„бһҡбһ бһјбһҸ! бһ”бһҫбһҷбһҫбһ„бһ…бһ„бҹӢбһ”бһ¶бһ“ JSON Exporter бһҷбһҫбһ„бһӮбҹ’бһҡбһ¶бһ“бҹӢбһҸбҹӮбһ”бһүбҹ’бһҮбһјбһ“ `JSONExportVisitor` бһҳбһҖбҹ” бһ”бһҫбһҷбһҫбһ„бһ…бһ„бҹӢбһ–бһ·бһ“бһ·бһҸбҹ’бһҷбһўбһҖбҹ’бһҒбһҡбһ¶бһңбһ·бһҡбһ»бһ‘бҹ’бһ’ бһҷбһҫбһ„бһ”бһүбҹ’бһҮбһјбһ“ `SpellCheckVisitor` бһҳбһҖ! бһҡбһ…бһ“бһ¶бһҹбһҳбҹ’бһ–бҹҗбһ“бҹ’бһ’бһҹбҹ’бһ“бһјбһӣбһ“бһ№бһ„бһҡбһҖбҹ’бһҹбһ¶бһ”бһ¶бһ“бһ“бһјбһңбһ—бһ¶бһ–бһ”бһҡбһ·бһҹбһ»бһ‘бҹ’бһ’ бһҹбҹ’бһ„бһ”бҹӢбһҹбҹ’бһ„бһ¶бһҸбҹӢ бһ“бһ·бһ„бһҹбһ»бһңбһҸбҹ’бһҗбһ·бһ—бһ¶бһ–бһ”бҹҶбһ•бһ»бһҸ!В»
* **бһҹбһјбһҖбҹ’бһҡбһ¶бһҸбҹ–** В«бһӣбҹ’бһўбһҺбһ¶бһҹбҹӢ бһӮбҹ’бһӣбһјбһҒбһ»бһ“бҹ” бһўбҹ’бһ“бһҖбһ”бһ¶бһ“бһҖбһ¶бһҡбһ–бһ¶бһҡбһҹбҹ’бһ“бһ¶бһҠбҹғбһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖ бһ“бһ·бһ„бһ”бһ¶бһ“бһҡбһҖбһғбһҫбһү Visitor Pattern бһ бһҫбһҷбҹ”В»

## бҹў. бһҹбҹҒбһ…бһҖбҹ’бһҸбһёбһҹбһ“бҹ’бһ“бһ·бһҠбҹ’бһӢбһ¶бһ“бһ“бҹғбһҹбҹ’бһҗбһ¶бһ”бһҸбҹ’бһҷбһҖбһҳбҹ’бһҳ (Architectural Conclusion)

The Visitor Pattern represents an operation to be performed on the elements of an object structure. It lets you define a new operation without changing the classes of the elements on which it operates, ensuring strict adherence to the Open-Closed and Single Responsibility principles.

Visitor Pattern бһҸбҹҶбһҺбһ¶бһ„бһұбҹ’бһҷбһ”бҹ’бһҡбһҸбһ·бһ”бһҸбҹ’бһҸбһ·бһҖбһ¶бһҡбһҠбҹӮбһӣбһҸбҹ’бһҡбһјбһңбһўбһ“бһ»бһңбһҸбҹ’бһҸбһӣбһҫбһ’бһ¶бһҸбһ»бһ•бҹ’бһҹбҹҶбһ•бҹ’бһҹбҹҒбһ„бҹ—бһ“бҹғбһҡбһ…бһ“бһ¶бһҹбһҳбҹ’бһ–бҹҗбһ“бҹ’бһ’ Objectбҹ” бһңбһ¶бһўбһ“бһ»бһүбҹ’бһүбһ¶бһҸбһұбҹ’бһҷбһўбҹ’бһ“бһҖбһ”бһ„бҹ’бһҖбһҫбһҸбһ”бҹ’бһҡбһҸбһ·бһ”бһҸбҹ’бһҸбһ·бһҖбһ¶бһҡбһҗбҹ’бһҳбһёбһҠбҹ„бһҷбһҳбһ·бһ“бһ”бһ¶бһ…бҹӢбһҖбҹӮбһ”бҹ’бһҡбҹӮ Class бһ“бҹғбһ’бһ¶бһҸбһ»бһ•бҹ’бһҹбҹҶбһ‘бһ¶бҹҶбһ„бһ“бҹ„бҹҮбһЎбһҫбһҷ бһҠбҹӮбһӣбһ’бһ¶бһ“бһ¶бһҖбһ¶бһҡбһўбһ“бһ»бһңбһҸбҹ’бһҸбһҷбҹүбһ¶бһ„бһҸбһ№бһ„бһҡбҹүбһ№бһ„бһ“бһјбһңбһӮбҹ„бһӣбһҖбһ¶бһҡбһҺбҹҚ Open-Closed бһ“бһ·бһ„ Single Responsibilityбҹ”
