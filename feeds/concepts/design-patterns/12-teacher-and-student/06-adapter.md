## ១. ទិដ្ឋភាពក្នុងថ្នាក់រៀន (Classroom Dialogue)

*(អាម៉ាប់កំពុងកាន់ពូថៅក្នុងថ្នាក់រៀន រៀបនឹងកាត់ខ្សែភ្លើង Laptop របស់ខ្លួន)*

**លោកគ្រូយក្ស (ស្រែកលាន់):** ហ្អើយ! អាម៉ាប់! ហែងចង់ធ្វើស្អី? ចង់កាប់សាលាចោលមែន?

**អាម៉ាប់:** អត់ទេលោកគ្រូ! ឌុយ Laptop ខ្ញុំវាជារន្ធសំប៉ែត តែព្រីភ្លើងសាលាជារន្ធមូល ដោតអត់ចូលសោះ! ខ្ញុំកំពុងតែកាត់ខ្សែវាចោល ដើម្បីតជើងថ្មីជារន្ធមូលអោយចូលលោកគ្រូ!

**លោកគ្រូយក្ស (ទាញយកពូថៅ):** អាភ្លើ! បើហែងកាត់វាចោល ពេលហែងយក Laptop ទៅដោតនៅផ្ទះហែងវិញ តើហែងកាត់វាម្តងទៀតមែនទេ? គេអត់ធ្វើអញ្ចឹងទេវើយ! នៅក្នុងការសរសេរកូដ បើប្រព័ន្ធពីរបញ្ចូលគ្នាអត់ចូល គេអត់ទៅរុះរើកូដចាស់ចោលទេ ព្រោះវាអាចនាំឱ្យរលាយប្រព័ន្ធ! គេត្រូវប្រើ **Adapter Pattern**!

**អាម៉ាប់ (អេះក្បាល):** អូ! Adapter ប្រែថាក្បាលព្រីបំប្លែងមែនទេគ្រូ?

**លោកគ្រូយក្ស:** អើហ្នឹងហើយ! ហែងទៅទិញក្បាលបំប្លែងមួយមក! ខាងម្ខាងវាមានរន្ធសំប៉ែតសម្រាប់ទទួលឌុយ Laptop ហែង ឯខាងម្ខាងទៀតវាមានជើងមូលសម្រាប់ដោតចូលព្រីសាលា! កម្មវិធីហែង (Laptop) មិនបាច់ដឹងទេថាព្រីសាលាជារន្ធអី វាគ្រាន់តែដឹងថាវាដោតចូលរន្ធសំប៉ែតរបស់ Adapter ជារឿងចប់! កូដហែងអត់ប៉ះពាល់មួយសរសៃ!

**អាម៉ាប់:** អូហូ! ឡូយមែនទែន! ចុះបើខ្ញុំយក Adapter ហ្នឹងទៅដោតនឹងព្រីអគ្គិសនី ហើយដោតចុងម្ខាងទៀតចូលក្នុងទុយោទឹកវិញ តើមានទឹកឬភ្លើងចេញមកលោកគ្រូ?

**លោកគ្រូយក្ស (គប់ហ្វឺតចំកណ្តាលថ្ងាស):** អាក្មេងរោគចិត្ត! Adapter គេប្រើសម្រាប់តែរបស់ដែលមានគោលដៅតែមួយទេ (គឺចម្លងភ្លើងដូចគ្នា) គ្រាន់តែខុស Interface! បើហែងយក Interface ដែលគ្មានទំនាក់ទំនងសោះមកកាត់តគ្នា កូដហែងនឹងចេញជា Frankenstein ហើយ! តោះ ចម្លងកូដនេះទៅ!

## ២. កូដនៅលើក្តារខៀន (Code on the Blackboard)

```java
// ១. Target (ព្រីសាលា ដែលចង់បានជើងមូល)
interface RoundSocket {
    void plugRound();
}

// ២. Adaptee (Laptop របស់អាម៉ាប់ ដែលជាជើងសំប៉ែត)
class FlatPlug {
    public void plugFlat() {
        System.out.println("ដោតចូលដោយជើងសំប៉ែត!");
    }
}

// ៣. Adapter (ក្បាលបំប្លែង ដែលអាម៉ាប់ទើបទៅទិញមក)
class FlatToRoundAdapter implements RoundSocket {
    private FlatPlug flatPlug;

    public FlatToRoundAdapter(FlatPlug flatPlug) {
        this.flatPlug = flatPlug;
    }

    @Override
    public void plugRound() {
        // នៅពេលព្រីសាលាត្រូវបានហៅ Adapter លួចយកក្បាលសំប៉ែតមកប្រើពីក្រោយខ្នង
        System.out.println("Adapter កំពុងបំប្លែងជើងមូលទៅជាជើងសំប៉ែត...");
        flatPlug.plugFlat(); 
    }
}
```

## ៣. ពិន័យ និងមេរៀន (Punishment & Lesson Learned)

**ពិន័យសម្រាប់អាម៉ាប់:**
- ត្រូវទៅទិញ Adapter ពិតប្រាកដមកដោតកុំព្យូទ័រ បើហ៊ានកាន់ពូថៅកាត់ខ្សែភ្លើងទៀត គ្រូនឹងកាត់ពិន្ទុអោយធ្លាក់!

**មេរៀនដែលទទួលបាន (Lesson Learned):**
- **បញ្ហា:** Interface កូដថ្មី និងកូដចាស់មិនត្រូវគ្នា ធ្វើឱ្យមិនអាចធ្វើការជាមួយគ្នាបាន ហើយការកែកូដចាស់គឺប្រថុយប្រថានពេក។
- **ដំណោះស្រាយ:** បង្កើត Adapter Class មួយដើម្បីដើរតួជាអ្នកបកប្រែកាត់កណ្តាល អនុញ្ញាតឱ្យពួកវាធ្វើការជាមួយគ្នាបានដោយមិនបាច់កែប្រែកូដដើមឡើយ។
