## ១. ទិដ្ឋភាពក្នុងថ្នាក់រៀន (Classroom Dialogue)

*(អាម៉ាប់កំពុងសរសេរកម្មវិធីបញ្ជាតេឡេទូរទស្សន៍ គេសរសេរ Logic ចុចបើកទូរទស្សន៍រាប់សិបបន្ទាត់ ញាត់ចូលទៅក្នុងកូដប៊ូតុងពណ៌ក្រហម! ស្រាប់តែលោកគ្រូយក្សដើរមកមើល)*

**លោកគ្រូយក្ស:** អាម៉ាប់! ហែងញាត់កូដ "បើកភ្លើងទូរទស្សន៍, បើកសំឡេង, ស្កេនប៉ុស្តិ៍" ចូលទៅក្នុងប៊ូតុងពណ៌ក្រហមធ្វើអី? ចុះបើអតិថិជនគាត់ចង់យកប៊ូតុងហ្នឹង ទៅបញ្ជាម៉ាស៊ីនត្រជាក់វិញ គាត់ត្រូវវះប៊ូតុងហ្នឹងចេញ ដើម្បីកែកូដខាងក្នុងមែនទេ?

**អាម៉ាប់:** លោកគ្រូ! ក៏ខ្ញុំចុចប៊ូតុងហ្នឹងដើម្បីបើកទូរទស្សន៍ អញ្ចឹងខ្ញុំសរសេរវាចូលប៊ូតុងហ្នឹងទៅ វាត្រូវហើយតើ! ប៊ូតុង និង ទូរទស្សន៍ វាជាប់គ្នាតែមួយ!

**លោកគ្រូយក្ស (គប់ហ្វឺត):** អាភ្លើ! ហ្នឹងគេហៅថា "Tight Coupling"! ប៊ូតុងមិនមែនជាអ្នកចេះបើកទូរទស្សន៍ទេ! វាគ្រាន់តែជាអ្នក "ចម្លងបញ្ជា"! គេត្រូវប្រើ **Command Pattern** វើយ! 
ហែងត្រូវសរសេរ Class មួយឈ្មោះ `TurnOnCommand` ជាកូនកញ្ចុះ! ពេលគេចុចប៊ូតុង ប៊ូតុងគ្រាន់តែទាត់គូទកូនកញ្ចុះនោះអោយរត់ទៅប្រាប់ទូរទស្សន៍ថា "វ៉ឺយ ថៅកែអោយបើក!" ទៅវាចប់ហើយ! ប៊ូតុងមិនបាច់ស្គាល់មុខទូរទស្សន៍ទេ ស្គាល់តែកូនកញ្ចុះហ្នឹងទៅបានហើយ!

**អាម៉ាប់ (អេះក្បាល):** អូហូ! ប៊ូតុងជាថៅកែ កូនកញ្ចុះជា Command! ចុះបើ... ថៅកែគាត់ចុចប៊ូតុងលឿនពេក ចុច ១០០ដង ក្នុង ១វិនាទី! កូនកញ្ចុះវាមិនរត់បុកគ្នាពេញផ្ទះទេអីលោកគ្រូ?

**លោកគ្រូយក្ស:** នេះហើយជាអំណាចរបស់ Command អាភ្លើ! ដោយសារតែ "បញ្ជា" ឥឡូវវាក្លាយជា Object ទៅហើយ យើងអាចយកវាទៅដាក់តម្រង់ជួរគ្នា (Queue) បាន! បើថៅកែចុច ១០០ដង យើងយកកូនកញ្ចុះ ១០០នាក់នោះអោយតម្រង់ជួរគ្នា ហើយដើរចូលទៅធ្វើការម្តងម្នាក់! ម៉ាស៊ីនអត់មានគាំងទេ!
ហើយបើថៅកែប្តូរចិត្ត គាត់ចុច "Undo" ហែងគ្រាន់តែប្រាប់កូនកញ្ចុះនោះថា "វ៉ឺយ ត្រលប់ក្រោយ! ធ្វើផ្ទុយពីអម្បាញ់មិញទៅ (`undo()`)"។ លឿនរហ័សទាន់ចិត្ត!

**អាម៉ាប់:** អូហូ! ចេះតម្រង់ជួរ ហើយចេះដើរថយក្រោយទៀត! ក្បាច់នេះអស្ចារ្យណាស់គ្រូ!

## ២. កូដនៅលើក្តារខៀន (Code on the Blackboard)

```java
// ១. Command Interface (កិច្ចសន្យាសម្រាប់កូនកញ្ចុះ)
interface Command {
    void execute(); // ដើរទៅមុខ
    void undo();    // ដើរថយក្រោយ
}

// ២. Receiver (ទូរទស្សន៍ - អ្នកត្រូវគេបញ្ជា)
class TV {
    public void turnOn() { System.out.println("TV is ON!"); }
    public void turnOff() { System.out.println("TV is OFF!"); }
}

// ៣. Concrete Command (កូនកញ្ចុះរត់ការ)
class TurnOnCommand implements Command {
    private TV tv;
    public TurnOnCommand(TV tv) { this.tv = tv; }
    
    public void execute() { tv.turnOn(); } // ធ្វើតាមបញ្ជា
    public void undo() { tv.turnOff(); }   // កែប្រែកំហុស
}

// ៤. Invoker (ប៊ូតុងថៅកែ)
class Button {
    private Command command;
    public void setCommand(Command c) { this.command = c; }
    public void click() { command.execute(); } // ទាត់គូទកូនកញ្ចុះអោយធ្វើការ
}
```

## ៣. ពិន័យ និងមេរៀន (Punishment & Lesson Learned)

**ពិន័យសម្រាប់អាម៉ាប់:**
- ត្រូវទៅឈរធ្វើជាកូនកញ្ចុះចាំរត់ការ (Command Object) អោយលោកគ្រូពេញមួយថ្ងៃ! លោកគ្រូចុចយកទឹក ត្រូវទៅយកទឹក! លោកគ្រូប្រាប់ថា Undo ត្រូវយកទឹកនោះទៅចាក់ចោលវិញ!

**មេរៀនដែលទទួលបាន (Lesson Learned):**
- **បញ្ហា:** ការសរសេរ Logic ភ្ជាប់ផ្ទាល់ទៅនឹង UI Element (ដូចជាប៊ូតុង) ធ្វើអោយពិបាកយកកូដទៅប្រើឡើងវិញ និងពិបាកធ្វើមុខងារ Undo/Redo ឬរៀបជាជួរ (Queue)។
- **ដំណោះស្រាយ:** បំប្លែង "បញ្ជា ឬសំណើ" អោយទៅជា Object (Command) ដែលផ្តាច់ចំណងរវាងអ្នកបញ្ជា (Invoker) និងអ្នកអនុវត្ត (Receiver) យ៉ាងមានប្រសិទ្ធភាព។
