## ១. ទិដ្ឋភាពក្នុងថ្នាក់រៀន (Classroom Dialogue)

*(ម៉ោង ៨:០០ ព្រឹក នៅសាលាវិទ្យាសាស្ត្រកុំព្យូទ័រ លោកគ្រូយក្សដើរចូលក្នុងថ្នាក់ដោយទឹកមុខក្រហមងាំង ហើយទះតុគ្រាំង!)*

**លោកគ្រូយក្ស:** អាម៉ាប់! ក្រោកឈរភ្លាម! ហែងប្រាប់អញមើល៍ ហេតុអីបានជាអញចុចប៊ូតុង Save ក្នុងកម្មវិធីហែងតែមួយវិនាទី កុំព្យូទ័រ Server សាលារបស់អញគាំងហុយផ្សែងចេញមកក្រៅអញ្ចឹង? ហែងសរសេរមេរោគចូលមែន?

**អាម៉ាប់ (ញ័រជើងទទ្រើក):** បា...បាទលោកគ្រូ! កូដខ្ញុំអត់មានមេរោគទេ! ខ្ញុំគ្រាន់តែសរសេរកូដឱ្យវាលឿន! ឱ្យតែលោកគ្រូចុច Save ម្តង ខ្ញុំវាយ `new DatabaseConnection()` មួយ ដើម្បីកុំឱ្យវាច្រណែនគ្នា។ លោកគ្រូចុចរាប់សិបដង វា `new` រាប់សិបដង... វាលឿនមែនទែនណាគ្រូ!

**លោកគ្រូយក្ស:** យី អាកូននេះ! (គ្រូគប់ហ្វឺតសរសេរក្តារខៀនទៅចំក្បាលអាម៉ាប់មួយផាំង!) លឿនរកខ្មោចថៅកែហែងអី! កុំព្យូទ័រអញគាំងព្រោះហែងហៅ Connection ច្រើនពេកស៊ីអស់ Memory ហ្នឹងឯង! ឯងដឹងទេថា ការបង្កើត Database Connection ម្តងៗ គឺស៊ីកម្លាំងម៉ាស៊ីនណាស់? គេត្រូវប្រើ **Singleton Pattern** វើយ! មានន័យថា "ទាញយក Connection តែមួយគត់" មកប្រើរួមគ្នាមួយភូមិ!

**អាម៉ាប់ (អេះក្បាល):** អូ! អញ្ចឹងខ្ញុំគ្រាន់តែបង្កើត Object មួយទុកចោលទៅ ហើយឱ្យគ្រប់គ្នាទាញយកទៅប្រើយ៉ាងម៉េចទៅលោកគ្រូ? ចុះបើមានក្មេងពាល វាយកពាក្យ `new` មកបង្កើត Connection ថ្មីលួចលាក់នៅក្រោយខ្នងលោកគ្រូ តើលោកគ្រូដឹងអត់?

**លោកគ្រូយក្ស:** ហែងគិតថាគ្រូភ្លើដូចហែង? ដើម្បីការពារកុំឱ្យអាពាលឯណាវាវាយពាក្យ `new` បាន គ្រូត្រូវចាក់សោរទ្វារផ្ទះ! គឺដាក់ `private constructor` វើយ! ពេលដាក់ `private` ហើយ មានតែ Class ហ្នឹងខ្លួនឯងទេទើបអាច `new` បាន។ បន្ទាប់មក គ្រូបង្កើតទ្វារ V.I.P មួយឈ្មោះ `getInstance()` ជាប្រភេទ `static` ឱ្យពួកហែងដើរចូលមកសុំប្រើ!

**អាម៉ាប់ (សើចកក្អិក):** ហាហា... ចុះបើសិស្សក្នុងថ្នាក់ ១០០នាក់ រត់សម្រុកចូលទ្វារ V.I.P ហ្នឹងព្រមគ្នាក្នុងវិនាទីតែមួយ (Race Condition) លោកគ្រូច្បាស់ជាវង្វេង ផលិត Object ១០០ ចែកឱ្យពួកខ្ញុំម្នាក់មួយមិនខាន!

**លោកគ្រូយក្ស (សម្លក់):** អេ៎! អាម៉ាប់នេះចេះគិតដល់ Multi-threading ទៀត! បើហ្វូងមនុស្សសម្រុកចូលមក គ្រូត្រូវប្រើក្បាច់ **Double-Checked Locking**! គ្រូឱ្យយាមកាមពីរជាន់។ ជាន់ទីមួយឆែកមើល បើជាន់គ្នា គ្រូចាក់សោរទ្វារ `synchronized` មួយជាន់ទៀត! ធានាថាមក ១ពាន់នាក់ព្រមគ្នា ក៏ចេញ Object តែមួយគត់មកដែរអាភ្លើ! អង្គុយចុះ ហើយចម្លងកូដនេះដាក់ចូលខួរក្បាលហែងទៅ!

## ២. កូដនៅលើក្តារខៀន (Code on the Blackboard)

```java
// សរសេរដោយលោកគ្រូយក្ស អមដោយស្នាមហ្វឺតក្រហមៗ
public class DBConnectionPool {
    
    // 1. volatile គឺដូចជាកាមេរ៉ាសុវត្ថិភាព ធានាថាគ្រប់ Thread មើលឃើញ Object ច្បាស់
    private static volatile DBConnectionPool instance;

    // 2. private constructor (ទ្វារដែកដែលអាម៉ាប់មិនអាចប្រើ new ទម្លុះបាន!)
    private DBConnectionPool() {
        System.out.println("កំពុងបង្កើត Connection ទំហំ 500MB... សុំរង់ចាំ!");
    }

    // 3. ទ្វារ V.I.P សម្រាប់ឱ្យហ្វូងមនុស្សមកសុំទាញយក
    public static DBConnectionPool getInstance() {
        // ឆែកជាន់ទី១ (ការពារកុំឱ្យរង់ចាំយូរ ពេលដែលវាមានស្រាប់ហើយ)
        if (instance == null) {
            // ចាក់សោរទ្វារ! ឱ្យចូលម្តងម្នាក់! (Synchronized Block)
            synchronized (DBConnectionPool.class) {
                // ឆែកជាន់ទី២ (ក្រែងលោមានអាណាមួយរត់ចូលមកទាន់ មុនពេលយើងចាក់សោរ)
                if (instance == null) {
                    instance = new DBConnectionPool();
                }
            }
        }
        return instance;
    }
}
```

## ៣. ពិន័យ និងមេរៀន (Punishment & Lesson Learned)

**ពិន័យសម្រាប់អាម៉ាប់:**
- ត្រូវទៅសរសេរពាក្យ `private constructor` និង `public static getInstance` ចំនួន ១០០០ ដង នៅលើក្តារខៀន ក្រោមពន្លឺថ្ងៃ!

**មេរៀនដែលទទួលបាន (Lesson Learned):**
- **បញ្ហា:** កម្មវិធីនឹងគាំង (Out of Memory) បើអ្នកបង្កើត Object ដែលស៊ីធនធានច្រើន ដដែលៗដោយមិនចាំបាច់ (ដូចជា Database Connection, File Manager, Hardware Controller)។
- **ដំណោះស្រាយ:** បិទសិទ្ធិមិនឱ្យគេប្រើ `new` ដោយដាក់ Constructor ជា `private`។ រួចប្រើប្រាស់ `static` method ដើម្បីចែកចាយ Object តែមួយគត់នោះទៅកាន់គ្រប់ទិសទីទូទាំងកម្មវិធី។
- **ចំណាំសំខាន់:** ការសរសេរ Singleton ត្រូវតែប្រយ័ត្នរឿង Multi-threading ជានិច្ច! បើមិនដូច្នេះទេ វានឹងបង្កើត Object លើសពីមួយនៅពេលចៃដន្យ (Race Condition)។
