# 🧩 Design Pattern — Explanation Strategies

Each subdirectory teaches the same set of design patterns (Singleton, Builder, Factory Method, and more) using a different pedagogical strategy.

| Directory | Teaching Strategy |
|:----------|:-----------------|
| [01-mit-professor](./01-mit-professor/) | Structured lecture — theory, formal definition, code |
| [02-feynman-technique](./02-feynman-technique/) | Explain it like a 12-year-old, then go deeper |
| [03-eli5](./03-eli5/) | Explain It Like I'm 5 — pure simplicity first |
| [04-analogy-bridge](./04-analogy-bridge/) | Real-world analogy that maps to the pattern |
| [05-socratic-method](./05-socratic-method/) | Questions that lead the reader to the answer |
| [06-journalist-inverted-pyramid](./06-journalist-inverted-pyramid/) | Most important fact first, context second |
| [07-storyteller-narrative-arc](./07-storyteller-narrative-arc/) | Story with a problem, conflict, and resolution |
| [08-engineer-requirements-constraints-solution](./08-engineer-requirements-constraints-solution/) | Requirements → constraints → solution |
| [09-pros-and-cons-compared](./09-pros-and-cons-compared/) | Side-by-side comparison across strategies |

See [explanation strategy guides](../../developer-habits/explanation-strategies/) for how each strategy works.
