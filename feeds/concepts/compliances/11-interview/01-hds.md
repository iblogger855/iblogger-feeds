## ១. កិច្ចសន្ទនាសម្ភាសន៍ (Interview Transcript)

**អ្នកសម្ភាសន៍ (Claire — Head of Security, French HealthTech):** Bonjour! សូមប្រាប់ខ្ញុំ — HDS គឺជាអ្វី? ពន្យល់ ១ ប្រយោគ ។

**បេក្ខជន (តារា — វិស្វករ Backend បទពិសោធន៍ ៤ ឆ្នាំ):** HDS ឬ Hébergeur de Données de Santé គឺជាវិញ្ញាបនប័ត្រជាកាតព្វកិច្ចរបស់ប្រទេសបារាំង សម្រាប់ស្ថាប័នដែលរក្សាទុកទិន្នន័យសុខភាពផ្ទាល់ខ្លួន តំណាងឲ្យអ្នកផ្តល់សេវាថែទាំសុខភាព។

**Claire:** ល្អ! ចុះប្រសិនបើយើងប្រើ AWS ដើម្បី Host កម្មវិធីសុខភាព ដែលដំណើរការទិន្នន័យអ្នកជំងឺបារាំង ហើយ AWS ក៏មានវិញ្ញាបនប័ត្រ HDS រួចហើយដែរ។ តើក្រុមហ៊ុនខ្ញុំត្រូវការសុំវិញ្ញាបនប័ត្រទៀតទេ?

**តារា:** **ពិតប្រាកដណាស់ — ជាដាច់ខាត** ។ HDS មានសកម្មភាពចំនួន ៦ ផ្សេងគ្នា។ AWS គ្របដណ្ដប់តែលើសកម្មភាពទី ១ ដល់ទី ៣ ប៉ុណ្ណោះ (គឺហេដ្ឋារចនាសម្ព័ន្ធជាក់ស្តែង, ការគ្រប់គ្រងហេដ្ឋារចនាសម្ព័ន្ធ និងប្រតិបត្តិការហេដ្ឋារចនាសម្ព័ន្ធ)។ ចំណែកឯសកម្មភាពទី ៤ (ការបង្ហោះកម្មវិធី), ទី ៥ (ការថែទាំកម្មវិធី) និង ទី ៦ (ការបម្រុងទុកទិន្នន័យ) គឺជាការទទួលខុសត្រូវរបស់ក្រុមហ៊ុនដែលដាក់ពង្រាយកម្មវិធី (Deploy Application) នោះ។ វិញ្ញាបនប័ត្ររបស់ AWS មិនគ្របដណ្ដប់លើស្រទាប់កម្មវិធី (Application Layer) ទេ។

**Claire:** អស្ចារ្យមែន! ចុះក្នុងចំណោមសកម្មភាពទាំង ៦ នេះ តើក្រុមហ៊ុនប្រភេទណាដែលត្រូវសុំវិញ្ញាបនប័ត្រសម្រាប់ទាំង ៦ តែម្តង?

**តារា:** គឺក្រុមហ៊ុនផ្តល់សេវា Data Center (On-premise) ដែលជាម្ចាស់ហេដ្ឋារចនាសម្ព័ន្ធទាំងអស់ — ដូចជាក្រុមហ៊ុន OVHcloud ឬ Scaleway ជាដើម។ ពួកគេត្រូវសុំវិញ្ញាបនប័ត្រសម្រាប់សកម្មភាពទី ១ ដល់ទី ៦។ ចំណែកឯក្រុមហ៊ុន SaaS ដែលជាវេទិកាសុខភាព ត្រូវសុំតែសម្រាប់សកម្មភាពទី ៤ ដល់ទី ៦ ប៉ុណ្ណោះ។ ក្នុងករណីប្រើប្រាស់ចម្រុះ (Hybrid): ក្រុមហ៊ុន IaaS ដូចជា AWS ត្រូវមានវិញ្ញាបនប័ត្រសម្រាប់សកម្មភាពទី ១-៣ រីឯក្រុមហ៊ុនកម្មវិធីវិញត្រូវមានវិញ្ញាបនប័ត្រសម្រាប់សកម្មភាពទី ៤-៦។

**Claire:** ពន្យល់បានក្បោះក្បាយល្អ! ចុះចំណែកច្បាប់កំណត់ទីតាំងទិន្នន័យវិញ? ប្រសិនបើកម្មវិធីរបស់យើងដាក់ឲ្យដំណើរការនៅសិង្ហបុរី ប៉ុន្តែអ្នកប្រើប្រាស់នៅប៉ារីស តើវាស្របច្បាប់ដែរឬទេ?

**តារា:** គឺ **ខុសច្បាប់** ។ លក្ខខណ្ឌតម្រូវរបស់ HDS បានចែងយ៉ាងច្បាស់ថា — ទិន្នន័យសុខភាពត្រូវតែរក្សាទុកនៅក្នុងប្រទេសបារាំង ឬក្នុងតំបន់សេដ្ឋកិច្ចអឺរ៉ុប (EEA)។ ប្រទេសសិង្ហបុរីគឺស្ថិតនៅក្រៅតំបន់ EEA។ ទោះបីជាមាន ISO 27001 ឬប្រព័ន្ធសុវត្ថិភាពរឹងមាំកម្រិតណាក៏ដោយ ប្រសិនបើ Server ស្ថិតនៅក្រៅប្រទេសបារាំងឬ EEA នោះវាផ្ទុយនឹងច្បាប់ HDS។

**Claire:** ចុះករណីអ្នកម៉ៅការបន្ត (Subcontractor) វិញ? ឧទាហរណ៍ ប្រសិនបើខ្ញុំប្រើសេវាគ្រប់គ្រងមូលដ្ឋានទិន្នន័យពីក្រុមហ៊ុនតតិយជន (Third-Party)?

**តារា:** អ្នកម៉ៅការបន្តក៏ចាំបាច់ត្រូវតែមានវិញ្ញាបនប័ត្រ HDS ផងដែរ។ លក្ខខណ្ឌតម្រូវ HDS បញ្ជាក់ថា — អ្នកម៉ៅការបន្តដែលដំណើរការឬរក្សាទុកទិន្នន័យសុខភាព ត្រូវតែមានវិញ្ញាបនប័ត្រសម្រាប់សកម្មភាពដែលពាក់ព័ន្ធនឹងខ្លួន។ កិច្ចសន្យាជាមួយអ្នកម៉ៅការបន្តត្រូវតែរួមបញ្ចូលនូវលក្ខខណ្ឌដូចជា៖ ទីតាំងទិន្នន័យ, សិទ្ធិធ្វើសវនកម្ម, ការជូនដំណឹងពេលមានឧប្បត្តិហេតុ និងសមត្ថភាពទាញយកទិន្នន័យ។

**Claire:** ល្អណាស់។ ចុះចំណែកការពិន័យវិញ — សូមប្រាប់ខ្ញុំបន្តិចមើល?

**តារា:** វាគឺជាទោសព្រហ្មទណ្ឌ។ សម្រាប់បុគ្គល — អាចជាប់ពន្ធនាគារ ៣ ឆ្នាំ។ សម្រាប់ស្ថាប័ន — អាចពិន័យជាប្រាក់ចំនួន ២២៥,០០០ អឺរ៉ូ។ បន្ថែមពីនេះ ទីភ្នាក់ងារ CNIL ក៏នឹងចូលមកស៊ើបអង្កេតករណីរំលោភបំពានច្បាប់ GDPR មាត្រា ៩ ផងដែរ — ដែលនេះជាការពិន័យទ្វេដង៖ ទាំងការបំពានច្បាប់ HDS និងការពិន័យតាមច្បាប់ GDPR (រហូតដល់ ៤% នៃចំណូលសរុបរបស់ក្រុមហ៊ុន)។

**Claire:** ចំពោះដំណើរការសុំវិញ្ញាបនប័ត្រវិញ — តើវាអាចចំណាយពេលប្រហែលប៉ុន្មានខែ?

**តារា:** ជាដំបូង ការមាន ISO 27001 គឺជាលក្ខខណ្ឌតម្រូវជាមុន។ សម្រាប់ពេលវេលាលឿនបំផុត (Best Case): ISO 27001 ចំណាយពេល ៦–៩ ខែ + ISO 27701 ២–៣ ខែ + សវនកម្ម HDS ២–៣ ខែ = សរុប ១០–១៥ ខែ។ ប្រសិនបើមាន ISO 27001 រួចហើយ នោះសវនកម្ម HDS នឹងចំណាយពេលត្រឹមតែ ២–៦ ខែប៉ុណ្ណោះ។ ស្ថាប័នចេញវិញ្ញាបនប័ត្រមានដូចជា៖ BSI, Bureau Veritas, LSTI, EY CertifyPoint។ វិញ្ញាបនប័ត្រមានសុពលភាព ៣ ឆ្នាំ និងតម្រូវឲ្យមានសវនកម្មតាមដានជារៀងរាល់ឆ្នាំ។

**Claire:** សំណួរចុងក្រោយ — សម្រាប់ប្រព័ន្ធផ្ញើសារផ្ទៃក្នុង (Internal Chat) ដែលក្រុមគ្រូពេទ្យប្រើប្រាស់។ តើសារទាំងនោះត្រូវការ HDS ដែរឬទេ?

**តារា:** វាអាស្រ័យលើស្ថានភាព។ ប្រសិនបើសារទាំងនោះមានផ្ទុកព័ត៌មានអ្នកជំងឺ (ដូចជា ការធ្វើរោគវិនិច្ឆ័យ, ការប្រើប្រាស់ថ្នាំ, ឬលទ្ធផលតេស្ត) — **គឺត្រូវការ**។ ព្រោះវាជាទិន្នន័យសុខភាពផ្ទាល់ខ្លួន។ ប៉ុន្តែប្រសិនបើសារទាំងនោះគ្រាន់តែជាការសម្របសម្រួលការងាររដ្ឋបាល (ដូចជាការកាលវិភាគប្រជុំ, កំណត់ហេតុរដ្ឋបាល) — **គឺប្រហែលជាមិនត្រូវការទេ**។ ជាគោលការណ៍រួម៖ ប្រសិនបើសារនោះអាចបញ្ជាក់ពីអត្តសញ្ញាណអ្នកជំងឺ និងទាក់ទងនឹងសុខភាព → វាត្រូវចាត់ទុកជាទិន្នន័យសុខភាព → ហើយតម្រូវឲ្យមាន HDS ។

**Claire:** (ញញឹម) ពន្យល់បានច្បាស់លាស់ណាស់។ ល្អ។ ខ្ញុំពិតជាពេញចិត្ត។

## ២. ការវាយតម្លៃរបស់អ្នកសម្ភាសន៍ (Interviewer's Evaluation)

✅ **ចំណុចខ្លាំង:**
- យល់ដឹងច្បាស់ពីការបែងចែកសកម្មភាពទាំង ៦ — ដែលនេះជាចំណុចដែលវិស្វករជាច្រើននៅខ្វះខាត។
- អាចឆ្លើយសំណួរបញ្ឆោតអំពី AWS បានភ្លាមៗ ដោយគ្មានការស្ទាក់ស្ទើរ។
- ដឹងច្បាស់ពីច្បាប់កំណត់ព្រំដែនតំបន់ EEA សម្រាប់ការផ្ទុកទិន្នន័យ។
- យល់ពីកាតព្វកិច្ចរបស់អ្នកម៉ៅការបន្ត។
- ដឹងពីកម្រិតនៃទោសព្រហ្មទណ្ឌ — ដែលបង្ហាញពីចំណេះដឹងផ្នែកច្បាប់កម្រិតខ្ពស់។

⚠️ **ចំណុចអាចកែលម្អ:**
- គួរតែបន្ថែមការលើកឡើងពីកាតព្វកិច្ចស្តីពីការប្រគល់ទិន្នន័យត្រឡប់មកវិញ (Data Portability) នៅពេលអតិថិជនឈប់ប្រើប្រាស់សេវាកម្ម។
- គួររំឭកបន្ថែមអំពីលក្ខខណ្ឌតម្រូវក្នុងកិច្ចសន្យា (contrat d'hébergement Clauses) ។

**លទ្ធផល: ✅ Hired (យល់ព្រមទទួលយក)** — ជាបុគ្គលិកផ្នែកសន្តិសុខដែលពិតជាមានជំនាញច្បាស់លាស់លើ HDS ។
