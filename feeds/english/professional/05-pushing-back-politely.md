# Pushing Back, Saying No, and Disagreeing Politely

**Tags:** #english #professional #communication #conflict  
**Read Time:** ~5 min

> You cannot be a good engineer if you cannot push back. But how you push back determines whether you change the outcome or just create conflict.

---

## The Structure of a Polite Pushback

```
[Acknowledge] + [Your concern] + [Evidence or reasoning] + [Alternative or question]
```

| Part | Purpose | Example |
|:-----|:--------|:--------|
| Acknowledge | Shows you heard them | `I understand the urgency...` |
| Concern | State your disagreement | `...but I'm worried that rushing this will introduce bugs.` |
| Evidence | Back it up | `The last time we skipped testing on a payment feature, we had a P1 incident within 24 hours.` |
| Alternative | Offer a path forward | `Could we do a smaller release with the core feature, and add the extras next sprint?` |

---

## Pushing Back on a Deadline

```
I understand you need this by Friday, but based on the scope,
I think we're looking at Monday at the earliest.

Here's the breakdown: [X takes 2 days, Y takes 1 day].
If we cut [Z], I could hit Friday. Would that work?
```

```
I want to be realistic with you — if I rush this to meet Friday,
I'm likely to introduce bugs that will cost more time to fix later.

I'd rather give you a reliable feature on Monday than a broken one on Friday.
Is that acceptable, or do we need to descope?
```

## Disagreeing with a Technical Decision

```
I see the thinking here, but I have a concern I'd like to raise
before we commit.

[State the concern clearly.] Have we considered [alternative]?
I think it might handle [edge case] better.

Happy to be wrong — could we discuss this briefly before we proceed?
```

```
I want to flag a potential issue with this approach.
[Explain the issue with specifics.]

I might be missing context — is there a reason we're not doing [alternative]?
```

## Saying No (or Not Yet)

### Saying no to extra work during a sprint
```
I'd love to help with this, but I'm at capacity right now with [current work].

Taking this on would mean pushing [X] to next sprint — is that acceptable?
If it's urgent, could someone else pick it up, or could we agree on priority?
```

### Saying no to a bad idea (diplomatically)
```
I have some reservations about this approach. My main concern is [X].

I don't want to shoot it down without understanding the full picture —
could you walk me through your thinking? There might be context I'm missing.
```

### Saying no to a request you don't have authority over
```
That's not my decision to make, but I can raise it with [person/team].
Would it help if I set up a quick call to discuss?
```

## Escalating a Disagreement

If a pushback in a 1-on-1 doesn't resolve it:

```
I want to make sure I'm not missing something — could we get
[TL/manager/stakeholder] involved to help us align?

I'm not trying to escalate — I just want to make sure we're making
this decision with full information on both sides.
```

## Softening Phrases

These words soften a statement without weakening your point:

| Hard version | Softer version |
|:------------|:--------------|
| `That's wrong.` | `I think there might be an issue with that.` |
| `We can't do that.` | `I'm not sure that's feasible given [constraint].` |
| `That will never work.` | `I have concerns about whether that will work in [specific scenario].` |
| `No.` | `I'm not in a position to do that right now because [reason].` |
| `You're making a mistake.` | `I'd like to flag a concern before we go further.` |

---

## Phrases That Work

### Starting a disagreement
- `I see it a bit differently...`
- `I want to raise a concern before we commit to this.`
- `Could I push back on this slightly?`
- `I might be wrong, but I'm worried that...`

### Holding your position under pressure
- `I hear you, but I still think [position] because [reason].`
- `I understand the urgency, but the risk here is [X].`
- `I want to make sure we're aligned on the trade-offs before we proceed.`

### Finding a middle ground
- `What if we [compromise position]?`
- `Could we try [approach] first and revisit if it doesn't work?`
- `Is there a version of this we can both agree on?`
