# Prepositions in Technical Writing

**Tags:** #english #daily #grammar #prepositions  
**Read Time:** ~5 min

> Prepositions are tiny words that native speakers use automatically. For everyone else, they require conscious attention — especially in technical contexts where the wrong one changes the meaning.

---

## deployed to vs deployed on vs deployed in

**deployed to** — the destination of a deploy
```
"The fix was deployed to production."          ✓
"The service is deployed to AWS."              ✓
```

**deployed on** — the platform or server it runs on
```
"The app is deployed on Kubernetes."           ✓
"The container runs on EC2."                   ✓
```

**deployed in** — a region or environment
```
"The service is deployed in us-east-1."        ✓
"We deploy in three regions."                  ✓
```

---

## depends on vs depends upon

Both are correct. `depends on` is more natural in everyday technical writing.
```
"This service depends on the auth service."    ✓
"This step depends on the migration being complete."  ✓
```

---

## integrates with vs integrates into

**integrates with** — works alongside, two-way relationship
```
"The app integrates with Stripe."              ✓
"Our system integrates with the hospital's EHR."  ✓
```

**integrates into** — becomes part of something larger
```
"This module integrates into the existing auth flow."  ✓
"The widget integrates into the checkout page."  ✓
```

---

## connect to vs connect with

**connect to** — a technical connection (database, server, API)
```
"The service connects to the database."        ✓
"Failed to connect to Redis."                  ✓
```

**connect with** — a human connection
```
"Let me connect you with the right person."    ✓
```

---

## listen on vs listen to

**listen on** — a port or address (technical)
```
"The server listens on port 3000."             ✓
```

**listen to** — audio, a person speaking
```
"Listen to what the client is asking for."     ✓
```

---

## write to vs write in vs write on

**write to** — output a destination
```
"The service writes to the database."          ✓
"Logs are written to stdout."                  ✓
```

**write in** — the language or format
```
"The script is written in Python."             ✓
"Write the config in YAML."                    ✓
```

**write on** — a physical surface
```
"Write the URL on the whiteboard."             ✓
```

---

## based on vs based in vs based at

**based on** — derived from, informed by
```
"The fee is calculated based on the transaction amount."  ✓
"Based on the logs, the error occurred at 14:32."         ✓
```

**based in** — where a company or team is located
```
"Our team is based in Phnom Penh."             ✓
```

---

## responsible for vs responsible to

**responsible for** — in charge of something
```
"The auth service is responsible for token validation."  ✓
"Dara is responsible for the payment integration."       ✓
```

**responsible to** — accountable to a person
```
"The team is responsible to the client."       ✓
```

## Quick Reference

| Phrase | Correct preposition | Example |
|:-------|:--------------------|:--------|
| deploy ___ production | to | deployed to production |
| runs ___ port 3000 | on | runs on port 3000 |
| written ___ TypeScript | in | written in TypeScript |
| depends ___ X | on | depends on the auth service |
| connect ___ the database | to | connects to the database |
| integrate ___ Stripe | with | integrates with Stripe |
| based ___ the requirements | on | based on the requirements |
| store ___ the database | in | stored in the database |
| save ___ a file | to | save to a file |
| error ___ line 42 | on | error on line 42 |
