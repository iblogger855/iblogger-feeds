# Commonly Confused Words in Technical Writing

**Tags:** #english #daily #grammar #writing  
**Read Time:** ~6 min

> These are the words that trip up non-native speakers (and many native speakers too). Getting them right makes your writing look professional.

---

## affect vs effect

**affect** = the action (verb) — to influence something  
**effect** = the result (noun) — the outcome

```
"The cache change will affect response times."      ✓ (verb)
"The cache change will effect response times."      ✗

"The effect of the cache change was faster responses."  ✓ (noun)
"The affect of the cache change was faster responses."  ✗
```

**Memory trick:** `A` for Action (affect), `E` for End result (effect)

---

## then vs than

**then** = time — what comes next  
**than** = comparison — measuring against something

```
"Deploy to staging, then run the tests."        ✓ (time)
"Deploy to staging, than run the tests."        ✗

"The new API is faster than the old one."       ✓ (comparison)
"The new API is faster then the old one."       ✗
```

---

## its vs it's

**its** = belonging to something (possessive)  
**it's** = it is (contraction)

```
"The service lost its connection to the database."   ✓ (belonging)
"The service lost it's connection to the database."  ✗

"It's a good approach for this use case."            ✓ (it is)
"Its a good approach for this use case."             ✗
```

**Test:** Replace with "it is". If it makes sense → use `it's`. If not → use `its`.

---

## your vs you're

**your** = belonging to you  
**you're** = you are

```
"Your PR is ready for review."     ✓
"You're PR is ready for review."   ✗

"You're going to need to rebase."  ✓
"Your going to need to rebase."    ✗
```

---

## there / their / they're

**there** = a place, or introducing something  
**their** = belonging to them  
**they're** = they are

```
"The bug is there, in the middleware."           ✓ (place)
"Their approach handles this edge case well."    ✓ (belonging)
"They're deploying to staging now."              ✓ (they are)
```

---

## ensure vs insure vs assure

**ensure** = make certain something happens  
**insure** = protect against risk (financial)  
**assure** = tell someone confidently to remove their doubt

```
"Please ensure the tests pass before merging."   ✓
"I can assure you the fix is correct."           ✓
"We need to insure the data centre equipment."   ✓ (financial insurance)
```

In technical writing you almost always want **ensure**.

---

## i.e. vs e.g.

**i.e.** (id est) = "that is" — gives the only explanation  
**e.g.** (exempli gratia) = "for example" — gives one of several examples

```
"Use a relational database, i.e. PostgreSQL."
(PostgreSQL IS the relational database you mean — only one)

"Use a relational database, e.g. PostgreSQL or MySQL."
(These are examples — there are others)
```

Both are followed by a comma. Both are lowercase.

---

## fewer vs less

**fewer** = countable things (you can count them individually)  
**less** = uncountable things (a continuous quantity)

```
"This query makes fewer database calls."    ✓ (you can count calls)
"This query makes less database calls."     ✗

"This approach uses less memory."           ✓ (memory is uncountable)
"This approach uses fewer memory."          ✗
```

---

## comprise vs compose

**comprise** = to consist of (the whole comprises the parts)  
**compose** = to make up (the parts compose the whole)

```
"The system comprises three services."           ✓ (whole → parts)
"Three services compose the system."             ✓ (parts → whole)
"The system is comprised of three services."     ✗ (common mistake — "comprised of" is not standard)
```

In practice, **"consists of"** avoids the confusion entirely:  
`"The system consists of three services."` ✓

---

## since vs because

**since** = from a point in time  
**because** = the reason why

```
"The bug has existed since the v2 migration."    ✓ (time)
"The test failed because the input was null."    ✓ (reason)
"The test failed since the input was null."      ✗ (ambiguous — avoid)
```

In technical writing, use **because** for causation. It's unambiguous.
