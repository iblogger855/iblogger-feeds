# Urgency and Priority Phrases

**Tags:** #english #idioms #urgency #priority #communication  
**Read Time:** ~4 min

> English-speaking tech teams have a whole vocabulary for expressing how urgent something is. Using the wrong level of urgency word either causes panic or gets ignored.

---

## Priority Levels

| Term | What it means | Example |
|:-----|:-------------|:--------|
| **P0** | Critical — everything stops until this is fixed. Production is down or data is at risk. | `"P0 — payment service is returning 500s for all users."` |
| **P1** | Severe — major feature broken, many users affected, fix today. | `"P1 — login is broken on iOS. Investigating."` |
| **P2** | Significant — important but has a workaround. Fix this sprint. | `"P2 — the export fails for files over 10MB."` |
| **P3** | Minor — low impact, can be scheduled. | `"P3 — button alignment off on the settings page."` |

---

## Common Urgency Phrases

### "Hair on fire"
The situation is extremely urgent — the metaphorical fire is immediate.

```
"We've got a hair-on-fire situation — the database migration 
 is running on production and it's been going for 2 hours."
```

### "Drop everything"
Stop what you're doing and focus on this immediately.

```
"Drop everything — there's a P0. Payment webhooks are failing."
```

### "This is a blocker"
Cannot proceed with other work until this is resolved.

```
"This is a blocker for the QA deployment — we need this fixed today."
"I'm blocked — can't continue until I get access to the staging environment."
```

### "Time-sensitive"
There's a deadline pressure, but it's not a crisis.

```
"This is time-sensitive — the client demo is on Friday."
```

### "We're on the clock"
There's a time limit — often used when customers are waiting or a deadline is approaching.

```
"We're on the clock — the client said they'll go to a competitor 
 if we can't deliver this by end of month."
```

### "ASAP"
As soon as possible. Urgent but not always P0.

```
"Can you review this ASAP? I need to deploy before the freeze."
```

Note: ASAP is overused. Be specific when you can: "by 3 PM today" is clearer than "ASAP."

---

## Escalation Phrases

When something needs to go to a higher level of authority:

```
"I need to escalate this — the client is threatening to cancel."
"This needs to go to engineering leadership — it's blocking the entire release."
"I'm escalating this incident to P0."
```

---

## Downgrading Urgency (When Others Are Panicking)

```
"Let's take a breath — this is serious but not production-down.
 We have time to think through the fix properly."

"I understand it feels urgent, but a hasty fix could make it worse.
 Let's take 15 minutes to diagnose properly."
```

---

## Setting Expectations on Timeline

When someone asks "when will this be done?":

| Answer type | Example |
|:------------|:--------|
| Specific | `"I'll have it done by 3 PM today."` |
| End of day | `"I can get this to you by EOD."` |
| This sprint | `"This sprint — I'll prioritise it tomorrow."` |
| After investigation | `"I need to investigate first. I'll have an ETA within the hour."` |
| Can't commit | `"I can't commit to a date right now without knowing the full scope."` |
