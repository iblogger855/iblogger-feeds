## ១. តើអ្វីជា ISO 27001? (What It Is)

ISO 27001 គឺជាស្តង់ដារអន្តរជាតិសម្រាប់ការបង្កើត ការអនុវត្ត ការថែទាំ និងការកែលម្អជាបន្តបន្ទាប់នូវ **ប្រព័ន្ធគ្រប់គ្រងសន្តិសុខព័ត៌មាន (ISMS)**។ ខុសពី SOC 2 (ដែលជារបាយការណ៍បញ្ជាក់ការអនុវត្ត) ISO 27001 ផ្ដល់ជូននូវ **វិញ្ញាបនបត្រ** ចេញដោយស្ថាប័នបញ្ជាក់វិញ្ញាបនបត្រដែលទទួលបានការទទួលស្គាល់ផ្លូវការ ដែលមានសុពលភាព ៣ ឆ្នាំ រួមជាមួយការធ្វើសវនកម្មត្រួតពិនិត្យប្រចាំឆ្នាំ។

ISO 27001 is the international standard for establishing, implementing, maintaining, and continually improving an **Information Security Management System (ISMS)**. Unlike SOC 2 (which is an attestation report), ISO 27001 results in a **certificate** issued by an accredited certification body — valid for 3 years with annual surveillance audits.

វាត្រូវបានវាយតម្លៃខ្ពស់ជាពិសេសនៅក្នុងតំបន់អឺរ៉ុប អាស៊ីប៉ាស៊ីហ្វិក មជ្ឈិមបូព៌ា និងលទ្ធកម្មសហគ្រាសទូទាំងសកលលោក។

It is particularly valued in Europe, Asia-Pacific, the Middle East, and enterprise procurement globally.

## ២. ប្រព័ន្ធគ្រប់គ្រងសន្តិសុខព័ត៌មាន (ISMS - Information Security Management System)

ប្រព័ន្ធគ្រប់គ្រងសន្តិសុខព័ត៌មាន (ISMS) គឺជាវិធីសាស្ត្រជាប្រព័ន្ធមួយក្នុងការគ្រប់គ្រងព័ត៌មានដែលងាយរងគ្រោះរបស់ក្រុមហ៊ុន ដើម្បីរក្សាព័ត៌មានទាំងនោះឱ្យមានសុវត្ថិភាព។ វាគ្របដណ្ដប់លើមនុស្ស ដំណើរការ និងបច្ចេកវិទ្យា។

An ISMS is a systematic approach to managing sensitive company information to keep it secure. It covers people, processes, and technology.

សមាសភាគស្នូល៖

Core components:

1. **បរិបទ (Context)** — ការស្វែងយល់អំពីស្ថាប័ន និងភាគីពាក់ព័ន្ធរបស់ខ្លួន
1. **Context** — understanding the organisation and its stakeholders
2. **ភាពជាអ្នកដឹកនាំ (Leadership)** — គោលនយោបាយសន្តិសុខ ការប្ដេជ្ញាចិត្តរបស់ថ្នាក់គ្រប់គ្រង
2. **Leadership** — security policy, management commitment
3. **ការរៀបចំផែនការ (Planning)** — ការវាយតម្លៃ និងការចាត់ចែងហានិភ័យ
3. **Planning** — risk assessment and treatment
4. **ការគាំទ្រ (Support)** — ធនធាន ការយល់ដឹង ការប្រាស្រ័យទាក់ទង
4. **Support** — resources, awareness, communication
5. **ប្រតិបត្តិការ (Operation)** — ការអនុវត្តការគ្រប់គ្រង
5. **Operation** — implementing the controls
6. **ការវាយតម្លៃលទ្ធផលការងារ (Performance evaluation)** — ការត្រួតពិនិត្យ សវនកម្ម ការពិនិត្យឡើងវិញដោយថ្នាក់គ្រប់គ្រង
6. **Performance evaluation** — monitoring, audit, management review
7. **ការកែលម្អ (Improvement)** — វិធានការកែតម្រូវ ការកែលម្អជាបន្តបន្ទាប់
7. **Improvement** — corrective actions, continual improvement

## ៣. ការគ្រប់គ្រងឧបសម្ព័ន្ធ A (Annex A Controls - ISO 27001:2022)

ISO 27001:2022 បានរៀបចំឧបសម្ព័ន្ធ A ឡើងវិញជា ៤ ប្រធានបទ ជាមួយវិធានការគ្រប់គ្រងចំនួន ៩៣ (កាត់បន្ថយពី ១១៤ ក្នុងឆ្នាំ ២០១៣)៖

ISO 27001:2022 reorganised Annex A into 4 themes with 93 controls (down from 114 in 2013):

| ប្រធានបទ (Theme) | វិធានការគ្រប់គ្រង (Controls) | វិស័យស្នូល (Key areas) |
|:---|:---|:---|
| **ស្ថាប័ន (Organisational)** (៣៧ វិធានការ) <br/> **Organisational** (37 controls) | គោលនយោបាយ តួនាទី ទំនាក់ទំនងដៃគូផ្គត់ផ្គង់ ការគ្រប់គ្រងឧប្បត្តិហេតុ <br/> Policies, roles, supplier relationships, incident management | |
| **មនុស្ស (People)** (៨ វិធានការ) <br/> **People** (8 controls) | ការពិនិត្យប្រវត្តិរូប ការបណ្ដុះបណ្ដាល លក្ខខណ្ឌការងារ <br/> Screening, training, terms of employment | |
| **រូបវន្ត (Physical)** (១៤ វិធានការ) <br/> **Physical** (14 controls) | តំបន់សុវត្ថិភាព ការការពារឧបករណ៍ <br/> Secure areas, equipment protection | |
| **បច្ចេកវិទ្យា (Technological)** (៣៤ វិធានការ) <br/> **Technological** (34 controls) | ការគ្រប់គ្រងការចូលប្រើប្រាស់ គ្រីបតូក្រាហ្វី ការគ្រប់គ្រងភាពងាយរងគ្រោះ ការត្រួតពិនិត្យ <br/> Access control, cryptography, vulnerability management, monitoring | |

វិធានការគ្រប់គ្រងថ្មីៗក្នុងឆ្នាំ ២០២២ (មិនមានក្នុងឆ្នាំ ២០១៣)៖

New controls in 2022 (not in 2013):

- ព័ត៌មានសម្ងាត់អំពីការគំរាមកំហែង (Threat Intelligence)
- Threat intelligence
- សន្តិសុខព័ត៌មានសម្រាប់សេវាកម្មក្លោដ (Cloud Services)
- Information security for cloud services
- ការត្រៀមខ្លួនផ្នែក ICT សម្រាប់អាជីវកម្មបន្ត (Business Continuity)
- ICT readiness for business continuity
- ការចម្រោះគេហទំព័រ (Web Filtering)
- Web filtering
- ការសរសេរកូដប្រកបដោយសុវត្ថិភាព (Secure Coding)
- Secure coding
- ការបិទបាំងទិន្នន័យ (Data Masking)
- Data masking
- ការការពារការលេចធ្លាយទិន្នន័យ (Data Leakage Prevention)
- Data leakage prevention
- ការត្រួតពិនិត្យសន្តិសុខរូបវន្ត (Physical Security Monitoring)
- Physical security monitoring
- ការគ្រប់គ្រងការកំណត់រចនាសម្ព័ន្ធ (Configuration Management)
- Configuration management
- ការលុបទិន្នន័យ (Deletion)
- Delete

## ៤. វិធីសាស្ត្រផ្អែកលើហានិភ័យ (Risk-Based Approach)

ISO 27001 មិនកំណត់វិធានការគ្រប់គ្រងជាក់លាក់ណាមួយឡើយ — វាទាមទារឱ្យអ្នក៖

ISO 27001 does not prescribe specific controls — it requires you to:

1. **កំណត់ទ្រព្យសកម្មព័ត៌មាន (Identify information assets)** — តើទិន្នន័យអ្វីខ្លះដែលអ្នកមាន?
1. **Identify information assets** — what data do you hold?
2. **វាយតម្លៃហានិភ័យ (Assess risks)** — តើអាចមានបញ្ហាអ្វីកើតឡើងខ្លះ? (ការគំរាមកំហែង × ភាពងាយរងគ្រោះ × ផលប៉ះពាល់)
2. **Assess risks** — what could go wrong? (threat × vulnerability × impact)
3. **ជ្រើសរើសវិធានការគ្រប់គ្រង (Select controls)** ដើម្បីកាត់បន្ថយហានិភ័យទាំងនោះ
3. **Select controls** to mitigate those risks
4. **ចងក្រងសេចក្ដីប្រកាសអំពីលទ្ធភាពអនុវត្ត (Statement of Applicability - SoA)** — តើវិធានការណាខ្លះដែលអ្នកអនុវត្ត និងហេតុអ្វី
4. **Document your Statement of Applicability (SoA)** — which controls you apply and why
5. **ទទួលយកហានិភ័យដែលនៅសេសសល់ (Accept residual risk)** — ការទទួលស្គាល់ជាផ្លូវការនូវហានិភ័យដែលនៅសល់
5. **Accept residual risk** — formally acknowledge remaining risks

## ៥. ដំណើរការទទួលបានវិញ្ញាបនបត្រ (Certification Process)

```
១. ការវិភាគគម្លាត (Gap Analysis)
   ប្រៀបធៀបស្ថានភាពបច្ចុប្បន្នធៀបនឹងតម្រូវការស្តង់ដារ ISO 27001

២. ការអនុវត្ត ISMS (ពី ៣ ទៅ ១២ ខែ)
   សរសេរគោលនយោបាយ អនុវត្តការគ្រប់គ្រង និងបណ្ដុះបណ្ដាលបុគ្គលិក

៣. សវនកម្មផ្ទៃក្នុង (Internal Audit)
   ការពិនិត្យឡើងវិញដោយឯករាជ្យផ្ទៃក្នុង មុនពេលសវនកម្មខាងក្រៅ

៤. ការពិនិត្យឡើងវិញដោយថ្នាក់គ្រប់គ្រង (Management Review)
   ថ្នាក់ដឹកនាំជាន់ខ្ពស់ពិនិត្យឡើងវិញនូវប្រសិទ្ធភាពរបស់ ISMS

៥. សវនកម្មដំណាក់កាលទី ១ (Stage 1 Audit - ការពិនិត្យឯកសារ)
   សវនករខាងក្រៅពិនិត្យមើលឯកសារគោលនយោបាយរបស់អ្នក

៦. សវនកម្មដំណាក់កាលទី ២ (Stage 2 Audit - សវនកម្មអនុវត្តជាក់ស្ដែង)
   សវនករធ្វើតេស្តថាតើការគ្រប់គ្រងនានាដំណើរការជាក់ស្ដែងដែរឬទេ

៧. ការចេញវិញ្ញាបនបត្រ (Certification Issued)
   មានសុពលភាព ៣ ឆ្នាំ

៨. សវនកម្មត្រួតពិនិត្យប្រចាំឆ្នាំ (Annual Surveillance Audits)
   ផ្ទៀងផ្ទាត់ការអនុលោមភាពជាបន្តបន្ទាប់

៩. សវនកម្មវាយតម្លៃឡើងវិញ (ឆ្នាំទី ៣)
   ការធ្វើសវនកម្មពេញលេញឡើងវិញ
```

```
1. Gap Analysis
   Compare current state against ISO 27001 requirements

2. ISMS Implementation (3–12 months)
   Write policies, implement controls, train staff

3. Internal Audit
   Independent internal review before external audit

4. Management Review
   Senior leadership reviews ISMS effectiveness

5. Stage 1 Audit (Documentation Review)
   External auditor reviews your documentation

6. Stage 2 Audit (Implementation Audit)
   Auditor tests whether controls work in practice

7. Certification Issued
   Valid 3 years

8. Annual Surveillance Audits
   Verify continued compliance

9. Re-certification Audit (year 3)
   Full re-audit
```

## ៦. ភាពខុសគ្នារវាង ISO 27001 និង SOC 2 (ISO 27001 vs SOC 2)

| ទិដ្ឋភាព (Aspect) | ISO 27001 | SOC 2 |
|:---|:---|:---|
| **ប្រភេទ** <br/> Type | វិញ្ញាបនបត្រ <br/> Certificate | របាយការណ៍សវនកម្ម <br/> Audit report |
| **វិសាលភាព** <br/> Scope | ISMS ពេញលេញ — ទូទាំងស្ថាប័ន <br/> Full ISMS — organisation-wide | លក្ខវិនិច្ឆ័យសេវាកម្មទុកចិត្តជាក់លាក់ <br/> Specific trust services criteria |
| **ភូមិសាស្ត្រ** <br/> Geography | ជាសកល (ជាពិសេស EU/APAC) <br/> Global (especially EU/APAC) | ជាចម្បងនៅសហរដ្ឋអាមេរិក <br/> Primarily USA |
| **រយៈពេល** <br/> Duration | វិញ្ញាបនបត្រ ៣ ឆ្នាំ + ត្រួតពិនិត្យប្រចាំឆ្នាំ <br/> 3-year certificate + annual surveillance | របាយការណ៍ Type II ប្រចាំឆ្នាំ <br/> Annual Type II |
| **ភាពបត់បែន** <br/> Flexibility | ផ្អែកលើហានិភ័យ — ជ្រើសរើសវិធានការផ្ទាល់ខ្លួន <br/> Risk-based — choose your controls | ផ្អែកលើលក្ខវិនិច្ឆ័យស្តង់ដារ <br/> Criteria-based |
| **កម្រិតនៃការណែនាំ** <br/> Prescriptiveness | មិនសូវកំណត់លម្អិត <br/> Less prescriptive | កំណត់លម្អិតជាង <br/> More prescriptive |
| **ការចំណាយ** <br/> Cost | ជាទូទៅពី £20K–£80K <br/> £20K–£80K typically | ជាទូទៅពី $30K–$80K <br/> $30K–$80K typically |

អតិថិជនសហគ្រាសជាច្រើនតម្រូវឱ្យមាន **ទាំងពីរ** — ISO 27001 សម្រាប់លទ្ធកម្មនៅតំបន់អឺរ៉ុប/អាស៊ីប៉ាស៊ីហ្វិក និង SOC 2 សម្រាប់លទ្ធកម្មនៅសហរដ្ឋអាមេរិក។

Many enterprise customers require **both** — ISO 27001 for EU/APAC procurement, SOC 2 for US procurement.

## ៧. អ្វីដែលត្រូវបង្កើត (What to Build)

```
ការអនុវត្តកញ្ចប់អប្បបរមាសម្រាប់ ISO 27001៖

□ គោលនយោបាយសន្តិសុខព័ត៌មាន — ទទួលបានការអនុម័តពីក្រុមប្រឹក្សាភិបាល
□ បញ្ជីសារពើភណ្ឌទ្រព្យសកម្ម — ចុះបញ្ជីរាល់ទ្រព្យសកម្មព័ត៌មានទាំងអស់
□ ការវាយតម្លៃហានិភ័យ — ចងក្រងជាឯកសារ វាយតម្លៃពិន្ទុ និងចាត់ចែង
□ សេចក្ដីប្រកាសអំពីលទ្ធភាពអនុវត្ត (SoA) — ឆ្លើយតបរាល់វិធានការទាំង ៩៣
□ គោលនយោបាយគ្រប់គ្រងការចូលប្រើប្រាស់ — ការផ្ដល់សិទ្ធិ ការពិនិត្យឡើងវិញ និងការលុបចោលសិទ្ធិ
□ គោលនយោបាយគ្រីបតូក្រាហ្វី — ទិន្នន័យណាដែលត្រូវកូដនីយកម្ម របៀបអនុវត្ត និងការគ្រប់គ្រងកូនសោ
□ នីតិវិធីគ្រប់គ្រងឧប្បត្តិហេតុ — ការរកឃើញ ការឆ្លើយតប និងការរាយការណ៍
□ ផែនការបន្តអាជីវកម្ម — កំណត់ RTOs និងធ្វើការសាកល្បង
□ ការវាយតម្លៃសន្តិសុខរបស់ដៃគូផ្គត់ផ្គង់ — សម្រាប់អ្នកផ្គត់ផ្គង់សំខាន់ៗ
□ កម្មវិធីសវនកម្មផ្ទៃក្នុង — ប្រចាំឆ្នាំ
□ ការពិនិត្យឡើងវិញដោយថ្នាក់គ្រប់គ្រង — ប្រចាំឆ្នាំ
□ ការបណ្ដុះបណ្ដាលការយល់ដឹងអំពីសន្តិសុខព័ត៌មាន — សម្រាប់បុគ្គលិកទាំងអស់ជារៀងរាល់ឆ្នាំ
□ ការគ្រប់គ្រងភាពងាយរងគ្រោះ — ស្កេន ដំឡើងបំណះ និងតាមដាន
□ វិធានការគ្រប់គ្រងសន្តិសុខរូបវន្ត — បន្ទប់ម៉ាស៊ីនបម្រើ គោលនយោបាយតុស្អាត (Clean Desk Policy)
```

```
Minimum ISO 27001 implementation:

□ Information security policy — board-approved
□ Asset inventory — all information assets catalogued
□ Risk assessment — documented, scored, treated
□ Statement of Applicability — all 93 controls addressed
□ Access control policy — user provisioning, review, revocation
□ Cryptography policy — what is encrypted, how, key management
□ Incident management procedure — detect, respond, report
□ Business continuity plan — RTOs defined and tested
□ Supplier security assessments — for critical vendors
□ Internal audit programme — annual
□ Management review — annual
□ Security awareness training — all staff, annually
□ Vulnerability management — scan, patch, track
□ Physical security controls — server room, clean desk policy
```
